# src/panels/ — Panel responsibilities

Each panel is a self-contained ImGui window. Panels receive references only to the state they need.

| Panel | Signature | Responsibility |
|-------|-----------|----------------|
| `doc_tabs_panel` | `DrawDocTabs(AppState&)` | Document tab bar — displays tab per open doc with basename or "Untitled", asterisk when unsaved, reorderable tabs; "+" button creates new blank document; clicking tab X signals close request via `close_doc_idx_requested` |
| `canvas_panel` | `DrawCanvas(AppState&)` | GL texture per doc, checkerboard, zoom-to-cursor, tool input, marching-ants overlay, floating selection move/scale, resize handles, layer-lock enforcement, track mouse position over canvas, handle close requests (with unsaved-changes modal) |
| `layers_panel` | `DrawLayers(CanvasState&)` | Layer list (top = highest index), add/delete/rename/visibility, layer thumbnail color swatch (`24.0f` × `24.0f`, scaled via `ui_scale::px()`) |
| `tools_panel` | `DrawTools(ToolsState&)` | Tool buttons (0=Brush 1=Eraser 2=Fill 3=Line 4=Rect 5=FilledRect 6=Circle 7=FilledCircle 8=Move 9=RectSelect 10=ColorPicker 11=ColorSelect) rendered as SVG textures via `icon_manager`; view toggles (symmetry, grid, onion skin); dual on/off icons for grid |
| `tool_settings_panel` | `DrawToolSettings(ToolsState&)` | Context-sensitive tool options: symmetry/onion/brush-size combos (widths via `ui_scale::px()`), brush shape toggle (Brush/Eraser only, button size SZ via `ui_scale::px(22 * 0.75)`), brush size only (Line), contiguous checkbox (Color Select only) |
| `palette_panel` | `DrawPalette(PaletteState&, SDL_Window*)` | "Color" window: current/previous swatch strip (swap button width `26.0f`, strip height `34.0f`, scaled via `ui_scale::px()`), HSV/RGB/HEX tabs (picker, HSV inputs, RGB inputs, HEX input with layout-aware width based on available region, recent row), palette grid (8-col, selected outline), Add/Remove/Sort; "edit" popup: rename field + **Import…** / **Export GPL** / **Export HEX** buttons (SDL3 async file dialogs via `palette_io`); Color window uses `ImGuiWindowFlags_NoScrollbar | ImGuiWindowFlags_NoScrollWithMouse` to prevent HEX tab flickering |
| `menu_bar` | `DrawMenuBar(AppState&, SDL_Window*, bool& show_log, bool& quit_requested)` → bool | File (New/Open/Save/Save As/Export > PNG / Sprite Sheet), Edit (Undo/Redo/Canvas Settings/Preferences), View (Log toggle); Preferences popup with UI Scale combo (75%/100%/125%/150%/200%, persisted via `ui_scale`), sprite sheet columns input (width via `ui_scale::px(80.0f)`); keyboard shortcuts Ctrl+S (save to current path or open dialog), Ctrl+N (new canvas), Alt+F4 (quit); unsaved-changes modal dialogs on quit/new if dirty; window title shows filename + "*" when dirty; SDL3 async file dialogs; detects PIXC vs PNG on open by magic bytes |
| `timeline_panel` | `DrawTimeline(CanvasState&)` | Transport buttons (|< prev, play/pause, >| next) advance frames within the active tag range (or all frames if `active_tag == -1`) at `cs.fps`; editable FPS via `DragInt`; playhead slider; animation tag combo + **Tags…** popup (add/rename/delete tags, DragInt start/end); horizontally-scrolling frame strip of 56×64 cards (live composite thumbnail, frame-number badge, active-border; cards outside tag range are dimmed); right-click context menu per card: **Duplicate** / **Delete**; add-frame card appends a new blank frame. Per-frame GL textures managed in `s_thumbs` (rebuilt each draw via `cs.composite_frame`). **active_layer safety**: on frame switch, `active_layer` is clamped to `frames[i].layers.size() - 1` so it never points past the target frame's layer stack; on new-frame creation, `active_layer` resets to 0 (new frames always start with exactly one layer). |
| `log_panel` | `DrawLog(bool* p_open)` | Displays `LogEntries()` ring buffer; auto-scrolls, Clear button; `p_open` wires the window close button to the View > Log toggle |
| `preview_panel` | `DrawPreview(const CanvasState&, bool* p_open)` | Floating mini-preview of the current frame (toggled via View > Preview, open by default). Window size is independent of zoom — user can resize freely. Independent zoom via scroll wheel (integer steps 1–32×, matching `canvas_panel` scheme: ±1 below 8×, multiples of 4 above — guarantees `W = cw * zoom` is always a whole pixel count, preventing sub-pixel edge bleed). Pan via middle-mouse drag or right-mouse drag (Mac-friendly). Origin snapped to integer screen coords (`std::round`) for pixel-perfect rendering. Checkerboard clipped to image bounds (not content area) to avoid bleeding one checker cell beyond the canvas edge. Uploads `cs.composite` every frame — no onion skin, no grid, no tool overlays. `p_open` wires the close button. Rendered last in `main.cpp` so it floats in front of docked panels. Uses `DrawCallback_SetSamplerNearest/Linear` for ImGui v1.92+ sampler fix. Window key `"Preview##pixcells"` (fresh ini slot). |

## canvas_panel internals
- **Per-document render state**: Each document has a `DocRenderState` struct with `texture` (main canvas GL texture), `onion_tex[2]` (prev/next frame GL textures), `tex_w/tex_h` (dimensions), `last_px` (mouse tracking), `was_painting` (stroke continuity flag), `drag` (`CanvasDragState`), and `prev_avail` (for deferred centering). Managed in a static vector `s_doc_render`. Textures are created when documents are opened and destroyed when they're closed; a shared `s_onion_buf` temporary buffer is used for onion-skin compositing.
- **Close request flow**: Ctrl+W sets `app.close_active_doc_requested = true`; canvas_panel's close-request handler checks this flag and any pending `close_doc_idx_requested` from the tab bar, then opens an "Unsaved Changes" modal if the target document is dirty. On modal confirmation, `close_doc(AppState&, idx)` frees GL resources and erases the document from `app.docs`. If the document count drops to zero, a blank document is created (invariant: always at least 1 doc open). `active_doc` is clamped to the remaining valid index range.
- Texture recreated when `cs.width()/height()` changes; updated via `glTexSubImage2D` when `cs.dirty`. Two additional `onion_tex[2]` GL textures (prev/next frame) are created alongside it and deleted/recreated on dimension changes.
- **Drag state per-doc**: Each `DocRenderState` holds a `CanvasDragState drag` struct with all shape, floating selection, and handle drag state (`shape_dragging`, `shape_sx/sy`, `float_drag_ox/oy`, `active_handle`, `handle_dragging`, `handle_start_x0/y0/x1/y1`), so drag interactions don't carry over when switching tabs.
- **Overlay and decoration drawing**: Canvas background (checkerboard), onion skin layers, and grid are drawn in `draw_canvas_overlays()`. Selection marching ants, floating selection pixel overlay, and resize handles are drawn in `draw_canvas_decorations()`. These functions separate the drawing concerns and keep `DrawCanvas()` more readable.
- **Symmetry**: when `tools.symmetry` is true, a magenta axis guide line is drawn over the canvas. `tools.symmetry_mode` controls which axis: 0=Horizontal (left-right mirror, vertical axis at x=W/2), 1=Vertical (top-bottom mirror, horizontal axis at y=H/2), 2=Both. All pixel-write operations (brush, eraser, flood fill, line/rect/ellipse shape commit) are mirrored via `sym_pt` (single point) and `sym_seg` (endpoint pair) lambdas defined just before the `IsItemHovered` block in `canvas_panel.cpp`. The shape preview `draw_px` lambda is also wrapped to mirror each rendered pixel, so both preview and commit stay in sync.
- **Onion skin**: when `tools.onion_skin` is true, `cs.composite_frame()` is called each render for adjacent frames and results uploaded to `onion_tex[0]` (prev) and `onion_tex[1]` (next). Which frames render is controlled by `tools.onion_skin_mode` (0=Both, 1=Previous only, 2=Next only); a `Combo` for this appears in `tools_panel` below the toggle when onion skin is on. Prev renders reddish `IM_COL32(255,128,128,128)`, next bluish `IM_COL32(128,128,255,128)`, both drawn via `dl->AddImage()` between the checkerboard and the active frame. Tags are not considered. The canvas_panel sets `tools.mouse_over_canvas` to track whether the mouse is inside the canvas window, which is used to determine whether to show tool-specific cursors or the default system cursor.
- **Sampler fix**: ImGui v1.92+ binds a LINEAR sampler before each draw. Inject `DrawCallback_SetSamplerNearest` / `DrawCallback_SetSamplerLinear` around `ImGui::Image()` to preserve `GL_NEAREST`.
- **Initial centering**: `cs.needs_center` triggers a one-time pan calculation to center the canvas in the panel. Centering is deferred until `GetContentRegionAvail()` returns the same value on two consecutive frames — on first launch (no `imgui.ini`), docking layout takes a few frames to settle and the available size is wrong until then. Each `DocRenderState` tracks `prev_avail` for this comparison, so each tab re-centers when first shown.
- **Zoom-to-cursor**: on scroll, adjusts `cs.pan` so the canvas pixel under the mouse stays fixed: `pan.x = mouse.x - (mouse.x - base.x - pan.x) / old_zoom * new_zoom - base.x`.
- Per-doc `was_painting` tracks stroke continuity; `push_snapshot()` fires once on stroke start (not per pixel).
- `base` = `ImGui::GetCursorScreenPos()` before `SetCursorScreenPos(origin)` — used as the fixed anchor for pan math.
- **Color picker (eyedropper, tool 10)**: picks the color at the mouse position, including transparent pixels. The picked RGBA value is converted via `ImGui::ColorConvertU32ToFloat4` and assigned to the active doc's `palette.primary_color`.
- **Color select (magic wand, tool 11)**: selects all pixels of the same color as the clicked pixel. When `tools.color_select_contiguous` is true, selects only connected pixels (flood-fill scan); when false, selects all matching pixels in the canvas. Creates a `SelectionState` with a per-pixel `mask` and bounding box returned by `raster::color_select()`; floating content is committed before starting a new color selection. The selection overlay renders each masked pixel's outline (marching-ants effect on cell boundaries).
- **Layer lock**: all pixel-write paths (brush, eraser, fill, shapes, cut/paste/delete) check `cs.active_layer_locked()` and log "Layer locked" without modifying the canvas.
- **HSV/RGB color picker alpha behavior**: The ImGui color picker with `ImGuiColorEditFlags_NoAlpha` flag still modifies the alpha channel when the user interacts with it. To preserve alpha when the user changes hue/saturation/value via the picker or HSV/RGB input fields, the picker saves `primary_color.w` before the picker call and restores it afterward (unless the picker returned true, meaning it was used — in which case alpha is set to 1.0f to ensure picked colors are opaque).
- **Selection overlay**: when `sel.active`, marching ants (alternating white/black at 8 Hz via `ImGui::GetTime()`) draw the selection boundary. For rectangular selections (empty mask), draws the bbox perimeter via two `AddRect` calls. For per-pixel selections (non-empty mask), draws the outline of each selected cell (edges where a selected pixel borders an unselected one or canvas edge) via individual `AddLine` calls.
- **Floating selection**: clicking inside an active selection with tool 9 calls `lift_selection()` (snapshot + erase pixels + store in `sel.float_pixels`); the lifted pixels render as `AddRectFilled` quads over the composite while floating, **clipped to the canvas** (`in_bounds` check per pixel) so the user sees exactly what survives commit. Mouse release keeps the float in place (it stays "in the air"); `commit_floating()` is called on tool switch or when the user clicks outside the float rect. Escape undoes the lift via `app.canvas.undo()` and clears `sel.active`.
- **Resize handles**: 8 `AddRectFilled` handles (TL/TM/TR/RM/BR/BM/BL/LM) drawn around `sel` when tool 9 is active, size ±`3.5f` scaled via `ui_scale::px()`. Dragging a handle resizes `sel.x0/y0/x1/y1`; on release, if floating, `nn_scale` (nearest-neighbour) scales `sel.float_pixels` to the new dimensions.
- **Selections may extend beyond the canvas**: `sel.x0/y0/x1/y1` are NOT clamped on create (release commit) or resize (handle drag) — the marquee/handles follow the mouse freely past the canvas edge. Off-canvas pixels are handled by the bounds-safe `Canvas::get`/`set`: reads (lift/copy/cut) return transparent, writes (paste/cut/delete/`commit_floating`/lift-erase) no-op, so only on-canvas content is affected. The floating overlay is clipped to the canvas so off-canvas content that will be dropped isn't shown.
- **Unified click handler**: tool 3–7 drag-start and the full tool-9 priority chain live in a single `if (mouse_in_win)` block, where `mouse_in_win` is a direct `GetWindowPos`/`GetWindowSize` screen-rect test (`IsWindowHovered` is unreliable when docked). This fires over the canvas image AND the surrounding margin, so drags can start/move/lift/resize off-canvas. The `IsItemHovered` branches for tools 3–7/9 are consolidated (tools 3–7 and tool 8/9 grouped into a single condition) to reset `was_painting`/`last_px` — they no longer start drags — so there is no double-handling.
- **Tool 9 click priority**: (1) handle hit → handle drag, (2) inside floating rect → re-drag float, (3) inside sel rect + unlocked → lift, (4) elsewhere → new selection drag (clears `sel.active`). No on-canvas bounds gate — all four fire whether the click is on- or off-canvas.
- **Raster algorithms**: the pure pixel-drawing routines (`paint_pixel`, `bresenham`, `flood_fill`, `draw_rect`, `draw_ellipse`, `nn_scale`, `rasterize_ellipse`) live in `src/raster.h/cpp` under `namespace raster`, operate on `Canvas&`, and have no ImGui/GL dependency (linked into `pix-cells-core`). canvas_panel calls them as `raster::fn(cs.active(), …)`.
- **Shape preview**: while `shape_dragging`, the preview overlay uses the same pixel-exact algorithms as the commit path (inline Bresenham + stamp for line, row/col loops for rect, the shared `raster::rasterize_ellipse` for circle/ellipse) — each canvas pixel renders as an `AddRectFilled` quad of size `cs.zoom`. `rasterize_ellipse` is a templated Bresenham ellipse-in-rect (Zingl) that emits pixels via a `plot` callback; both the preview (`draw_px` lambda) and `raster::draw_ellipse` commit (`c.set` lambda) call it, so they can't drift. It fits the drag bounding box exactly (odd & even diameters) and span-fills the filled variant from the same edge points as the outline. Tool 9 (select) keeps a vector marching-ants overlay instead. Holding Shift during a rect/circle drag constrains the bounding box to a square (applied to both preview and commit via `epx`/`epy` effective endpoint).

## Icons
Icons are SVG files in `icons/` rasterized at runtime via lunasvg. `icon_manager::init("icons")` is called in `main.cpp` after `ImGui_ImplOpenGL3_Init`; `icon_manager::shutdown()` is called before `ImGui_ImplOpenGL3_Shutdown`. Panels call `icon_manager::get("name")` to retrieve an `ImTextureID` (GL texture, 32×32 RGBA, lazily loaded and cached). All icon buttons use `ImGui::ImageButton` with `ImGuiStyleVar_FramePadding` set to `(FP, FP)` (tools panel uses `FP=4` with `ICON=22`). Missing SVG files return texture ID 0 (blank button, no crash). SVG files use the naming convention `name.svg` / `name_filled.svg` / `name_off.svg` / `name_open.svg`.

SDL cursors are managed by `cursor_manager` (`src/cursor_manager.h/cpp`). `set_for_tool(tool_index, mouse_pressed, mouse_over_canvas)` is called each frame in `main.cpp`. When `mouse_over_canvas` is false, uses the default system cursor; when true, Move tool (8) uses `pan_tool.svg` / `pan_tool_alt.svg` depending on `mouse_pressed`; Color Picker (10) uses `eyedropper.svg`; all others use `point_scan.svg`.

## Adding a new tool
Tool indices are named constants in the `tool::` namespace in `app_state.h` (`tool::Brush`, `tool::RectSelect`, …) with a `tool::is_shape(t)` helper — use these instead of bare integers.
1. Add a constant to the `tool::` namespace in `app_state.h` (and extend `is_shape()` if the new tool is a shape).
2. Add an entry to `tool_defs[]` in `tools_panel.cpp` with the SVG icon name, `##tN` id, and tooltip. Place the corresponding `name.svg` in `icons/`.
3. Add a handler branch in `canvas_panel.cpp`:
   - Brush-like tools: inside `IsItemHovered` in the `else` branch (these only act over the canvas image).
   - Shape tools (click-drag) and selection: drag-START lives in the **unified `if (mouse_in_win)` block** (a `GetWindowPos`/`GetWindowSize` screen-rect check), so drags can begin over the image OR in the margin. The matching `IsItemHovered` branch is reduced to just resetting `was_painting`/`last_px` (so the tool doesn't fall through to the brush `else`). Extend the `active_tool >= tool::Line && active_tool <= tool::FilledCircle` range there; commit in the `shape_dragging` `else` block's switch (cases use `tool::` constants and call the `raster::` drawing functions — `raster::bresenham`/`draw_rect`/`draw_ellipse` — on `cs.active()`); add a pixel-exact preview branch in `if (shape_dragging)` using `draw_px` / `stamp` lambdas (see the existing `tool::Line`/`Rect`/`FilledRect`/`Circle`/`FilledCircle` branches).
   - Selection-like tools (point-select with instantaneous result): add an `else if` branch in the unified `mouse_in_win` block on `IsMouseClicked(ImGuiMouseButton_Left)`; call the raster algorithm (e.g., `raster::color_select()`) to populate `SelectionState` fields, or `sel.active = false` if no pixels matched. Do not call `rebuild_composite()`.
4. If the tool modifies `SelectionState`, ensure all selection-clearing code paths (Escape, click-outside, tool-switch) also clear `sel.mask`.
5. Update the `tool::Move` and `tool::RectSelect` checks in `canvas_panel.cpp` if inserting before them.
6. If the tool has settings (brush size, contiguous mode, etc.), add a `bool show_<setting>` flag and checkbox in `DrawToolSettings()`.
7. Add a keyboard shortcut in `input_handler.cpp` if desired (assign `tool::` constants).
